Create `tests/test_middleware/test_request_id.py` with the following test class and methods:

1. Import `pytest` and `from flask import Flask`.

2. Create class `TestRequestIDMiddleware`:
   - `test_request_id_generated(client)`: Send GET to `/health`. Assert the response contains `X-Request-ID` header.
   - `test_request_id_returned_in_response_headers(client)`: Send GET to `/health`. Assert the `X-Request-ID` response header value is a non-empty string (8 characters, as set by `generate_request_id`).
   - `test_request_id_included_in_error_response(client)`: Send POST to `/v1/chat/completions` with an empty JSON payload `{}`. Assert the error response (400) contains a `request_id` field in the JSON body. Use `mock_openwebui_client` which runs autouse to avoid real backend calls.
   - `test_custom_request_id_respected(client)`: Send GET to `/health` with a custom `X-Request-ID` header set to `"custom-id-456"`. Assert the response `X-Request-ID` header equals `"custom-id-456"`.
   - `test_request_id_in_logs(client, caplog)`: Send GET to `/health`. Using `caplog` from pytest, set `caplog.set_level("INFO")`, then assert that at least one log record contains the request_id value found in the response header.

Use the existing `client` fixture from `conftest.py` for all tests.
</action>
